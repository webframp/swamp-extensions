# Cloudflare Audit Extension Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a Cloudflare security and configuration audit that gathers zone settings, DNS records, WAF rules, worker config, and cache settings, then generates a severity-rated report with actionable findings.

**Architecture:** Workflow + report extension following the SRE health check pattern. Parallel jobs gather data from 5 existing Cloudflare models, a finalize job gates report generation, and a workflow-scope report analyzes all step outputs against security and hygiene thresholds.

**Tech Stack:** TypeScript (Deno), swamp workflow YAML, `@webframp/cloudflare` models (zone, dns, waf, worker, cache)

## Workflow Design

**Inputs:**
- `zoneId` (string, required) — Cloudflare zone ID to audit

**Jobs:**

### Job 1: `zone-config` (Zone settings and metadata)

| Step | Model | Method | Inputs |
|------|-------|--------|--------|
| `list-zones` | cf-zone | list | — |
| `get-settings` | cf-zone | get_settings | zoneId |

### Job 2: `security` (WAF and firewall posture)

| Step | Model | Method | Inputs |
|------|-------|--------|--------|
| `list-rules` | cf-waf | list_rules | — |
| `list-packages` | cf-waf | list_packages | — |
| `security-events` | cf-waf | get_security_events | limit=100 |

### Job 3: `dns-and-edge` (DNS, workers, cache)

| Step | Model | Method | Inputs |
|------|-------|--------|--------|
| `list-dns` | cf-dns | list | — |
| `list-workers` | cf-worker | list_scripts | — |
| `list-routes` | cf-worker | list_routes | zoneId |
| `cache-settings` | cf-cache | get_settings | — |
| `cache-analytics` | cf-cache | get_analytics | — |

### Job 4: `finalize` (Gate for report)

Depends on jobs 1-3 with `condition: completed`. Runs a single step:

| Step | Model | Method | Inputs |
|------|-------|--------|--------|
| `zone-detail` | cf-zone | get | zoneId |

All steps use `allowFailure: true`.

## Report: Audit Checks

### Security

| Check | Source | Threshold | Severity |
|-------|--------|-----------|----------|
| SSL mode | zone settings `ssl` | `full`/`strict` = ok, `flexible` = warn, `off` = critical |
| Always Use HTTPS | zone settings `always_use_https` | `on` = ok, `off` = warn |
| WAF rules present | waf list_rules | >= 1 active rule = ok, 0 = warn |
| WAF packages active | waf list_packages | >= 1 = ok, 0 = warn |
| Paused firewall rules | waf list_rules | any paused = warn per rule |
| Security events volume | waf get_security_events | informational summary |

### DNS Hygiene

| Check | Source | Threshold | Severity |
|-------|--------|-----------|----------|
| Unproxied A/AAAA/CNAME | dns list, proxied=false | warn per record |
| Dangling CNAMEs | dns list, CNAME content matches takeover patterns | critical per record |
| Missing CAA records | dns list, no CAA type | warn |

### Edge/Workers

| Check | Source | Threshold | Severity |
|-------|--------|-----------|----------|
| Workers.dev subdomain | worker list_scripts (inferred) | warn if discoverable |
| Workers without routes | worker list_scripts vs list_routes | warn per orphaned script |

### Cache/Performance

| Check | Source | Threshold | Severity |
|-------|--------|-----------|----------|
| Development mode enabled | zone settings `development_mode` | on = critical |
| Cache level | cache get_settings `cacheLevel` | `bypass`/`basic` = warn |
| Low cache hit rate | cache get_analytics | < 50% = warn |

### Zone

| Check | Source | Threshold | Severity |
|-------|--------|-----------|----------|
| Zone paused | zone list/get `paused` | true = critical |
| Zone status pending | zone list/get `status` | != active = warn |

## Report Output

**Markdown:** Status banner, critical issues, warnings, all-checks table, recommendations list (same layout as SRE health report).

**JSON:** `{ workflowName, workflowStatus, timestamp, zone, findings[], overallStatus, recommendations[] }` plus raw data sections (zoneSettings, dnsRecords, wafRules, etc.) for programmatic consumption.

**Overall status rollup:** any critical = CRITICAL, any warn = WARNING, all errors = DEGRADED, else HEALTHY.

## Testing

- Report tests use `createReportTestContext` with mock `stepExecutions` and `dataRepository`
- Test scenarios: all-healthy, mixed warnings, critical SSL, dangling CNAME, dev mode on, zone paused
- No live Cloudflare API calls in tests
